1.提示，警告

  1. 提示(//MARK相当于#pragma mark)

  2. 警告(//TODO:或者//FIXME:相当于#warning message)

    中间不加”-“会在同级下，加”-“在不同一级下
//MARK: 1           //MARK: - 1
//MARK: 2            //MARK: - 2

2.AppDelegate代码结构

  1. @UIApplicationMain就是程序的入口
  2. 只有.swift没有.h / .m在 Swift 中默认全局共享
  3. 所有的代码 都包装在{ }, 默认方法都有一个缩进！再也没有end

3.视图创建

UIView

let v0 = UIView.init(frame: CGRect(x: 0, y: 80, width: 100, height: 100))
let v1 = UIView(frame: CGRect(x: 0, y: 180, width: 100, height: 100))
let v2 = UIView()

注:

  1. view == self.view (view.addSubview(v))

    即访问当前对象属性,可以不使用 'self.'
建议:都不用,在编译器提示的时候, 再添加, 会对'语境'有更好体会
原因:闭包(类似于 OC block),需要使用 'self.'

  2. 类名() == alloc / init 等价

  3. 设置CGRect:打出 CGRect(会自动联想
  4. 设置颜色 UIColor.red或者.red或者 输入color联想到 Color Literal，回车即可
  5. 没有;,目的是分隔语句, 在swift中默认不需要,
    只有当`let a = 10; let b = 20`类似时才加';'

UIButton

let btn = UIButton(type: .contactAdd)
btn.addTarget(self, action: #selector(ClickMe), for: .touchUpInside)


@objc func ClickMe() -> () {
    print(#function)
}

注：

  1. 枚举类型

    .contactAdd == UIButtonTypeContactAdd

    .custom == UIButtonTypeCustom

  2. 监听方法

    #selector == @selector
    #selector(ClickMe) 如果带参数，不需要使用:

  3. 调试

    print() == NSLog()

    打印时没有时间；效率比NSLog高

    #function == __FUNCTION__

4.语法

常量(let)/变量(var)

  1. 常量 let, 定义后，不能修改

    变量 var, 定义后，可以修改

    关于var和let的选择：不可变的会更安全,开发的时候,通常都先使用let,在需要变化的时候再改成var

  2. 自动推导,变量/常量的类型会根据右侧的代码执行结果,推导对应的类型

    热键:opition + click(查看类型) 

      func demo() {
    var x = 10 //(整数,Int(相当于NSInteger))
    x = 20
    let y = 10.5 //(小数,Double精度高(相当于CGFloat))
    //修改视图属性
    let v = UIView()
    //仅仅修改的是v的属性,并没有修改v的指针地址
    v.backgroundColor = UIColor.red
    print(x)
    print(y)
    print(v)
}

  3. 在Swift中对类型要求异常严格

    3.1 任何不同类型的数据之间，不允许直接运算；

    3.2 不做任何默认的隐式的转换

    3.3 所有的类型确定,都要有程序员负责

    3.4 在Swift中，不存在基本数据类型，都是结构体！

    3.5 如果需要指定变量/常量的类型,也可以直接使用let x: 类型 = 值，在Swift开发中,极少使用直接指定类型,通常都是自动推导。

    func demo1()  {
    let x = 10
    let y = 10.5
    //1.将 y 转换成 Int
    //OC 中的写法(Int)y => 类型强转
    //Swift 中 Int() '结构体'构造函数
    print(x + Int(y))
    //2.将 x 转换成 Double
    print(Double(x) + y)
}
func demo2() {
    //如果需要指定变量/常量的类型,也可以直接使用 let x: 类型 = 值
    //提示:在Swift开发中,极少使用直接指定类型,通常都是自动推导!
    let x: Double = 10
    let y = 10.5
    print(x + y)
}
 func demo3() {
        // Variable 'x' was never mutated; consider changing to 'let' constant
        //变量 x 从来没有被修改,建议改为 'let'
        var x = 10
        let y = 10
        print(x + y)
}

可选项(Optional)

//最常见错误
//Unexpectedly found nil while unwrapping an Optional value
//在解包的时候发现了 nil 值 
//定义可选项 '?'
//解包 '!' 准备计算
func demo() {
    //1.原始的可选项定义
    //none 没有值 some 某一类值
    let x: Optional = 10;

    //2.简单定义
    // `?`用来定义 y 是个可选的 Int 类型,可能没有值,也可能有一个整数
    let y: Int? = 20

    //let y: Int? //= 20
    //Definition conflicts with previous value
    //常量 y 使用之前必须初始化

    //let y: Int? = nil

    //输出结果Optional(10),提示这是一个可选值
    print(x!)
    print(y!)
    //1.不同类型之间的值不能直接运算!如果没有值是 nil 不是任何数据类型,不能参与计算
    //print(x + y)
    //2.'!'强行解包：从可选值中强行获取对应的非空值，如果真的是 nil,就会崩溃
   //程序员必须对每一个'!'负责
   //程序中少用'!'
    print(x! + y!)
}

func demo1() {
    //1.定义 y,没有初始化
    let y: Int?
    //2.给常量设置数值,初始化的工作能做一次
    y = 20
    //3.再次修改常量会报错
    //y = 30
    print(y!)

    //var 的可选值默认为 nil
    //let 的可选值没有默认值
    var x: Int?
    x = 100
    x = 10
    print(x)
}

分支(if)/三目(var)

//MARK:简单的分支
func demo()  {
    let x = 10
    /**
     1.条件不需要 ()
     2.语句必须有 {}
      - 很多h公司代码规范中禁止不使用 {}
    */
    //if x > 5 print("大了") 不允许
    if x > 5 {
        print("大了")
    } else {
        // Will never be executed
        //永远不会被执行 - 编译器编译的时候,会做语法检查!
        print("小了")
    }
}

//MARK:三目
func demo1()  {
    let x = 3
    //三目语法和 OC 一样的
    x > 5 ? print("大了") : print("小了")
    //'()' 表示空执行
    x > 3 ? print("大了") : ()
}
